# 动态规划算法学习内容摘要

## 一、核心概念
动态规划（Dynamic Programming，DP）是运筹学的一个分支，用于求解多阶段决策过程的最优化问题。由美国数学家贝尔曼（R.Bellman）于20世纪50年代初提出。其基本思想是将复杂问题分解为相互重叠的子问题，通过保存子问题的解（填表）避免重复计算，以空间换时间。

## 二、基本思想
1. **最优子结构**：问题的最优解包含其子问题的最优解。
2. **重叠子问题**：子问题被重复计算多次，通过记忆化存储避免重复。
3. **无后效性**：未来状态只依赖于当前状态，与过去状态无关。
4. **多阶段决策**：将问题划分为若干相互联系的阶段，每个阶段做出决策。

## 三、适用场景
1. **最优化问题**：求最大值、最小值、最优方案等。
2. **计数问题**：求方案总数、路径数等。
3. **可行性问题**：判断是否存在满足条件的解。
4. **典型特征**：问题具有最优子结构和重叠子问题性质。

## 四、经典问题
1. **背包问题**
   - 0/1背包：每种物品选或不选。
   - 完全背包：每种物品无限个。
   - 多重背包：每种物品有限个。
2. **最长公共子序列（LCS）**：求两个序列的最长公共子序列长度。
3. **最长递增子序列（LIS）**：求序列中最长递增子序列长度。
4. **最短路径问题**（如Floyd算法）。
5. **矩阵链乘法**：求矩阵相乘的最小计算代价。
6. **编辑距离**（Levenshtein距离）：求两个字符串的最小编辑操作数。
7. **最大子段和**：求连续子数组的最大和。
8. **硬币找零**：求凑成金额的最少硬币数。
9. **股票买卖问题**（含冷冻期、手续费等变种）。
10. **打家劫舍系列**（相邻限制、环形等变种）。

## 五、基本步骤
1. **定义状态**：明确dp数组的含义，通常表示某个子问题的解。
2. **确定状态转移方程**：找出状态之间的关系式（递推公式）。
3. **初始化边界条件**：设置最小子问题的初始值。
4. **确定计算顺序**：通常自底向上（迭代）或自顶向下（递归+记忆化）。
5. **返回结果**：根据dp数组得出最终答案。

## 六、学习路径建议
1. **入门阶段**（理解概念与简单问题）
   - 学习基本思想、最优子结构、无后效性等概念。
   - 练习经典入门题：斐波那契数列（记忆化递归）、爬楼梯、最小路径和等。
2. **基础阶段**（掌握常见模型）
   - 0/1背包及其变种（416分割等和子集、494目标和）。
   - 最长公共子序列、最长递增子序列。
   - 编辑距离、最大子段和。
3. **进阶阶段**（复杂模型与优化）
   - 完全背包、多重背包、混合背包。
   - 状态压缩DP（如旅行商问题）。
   - 区间DP（如石子合并、括号匹配）。
   - 树形DP（如二叉树中的最大路径和）。
4. **实战阶段**（综合应用与竞赛题）
   - LeetCode/牛客网动态规划专题训练。
   - 参加算法竞赛（如ACM、蓝桥杯）中的DP题目。
   - 阅读经典教材《算法导论》动态规划章节。
5. **资源推荐**
   - 书籍：《算法导论》《算法竞赛入门经典》。
   - 在线课程：中国大学MOOC《算法设计与分析》。
   - 刷题平台：LeetCode、牛客网、AcWing的DP专题。

## 七、注意事项
1. 注意区分贪心算法与动态规划的适用条件（贪心要求局部最优能导致全局最优）。
2. DP的空间优化技巧（如滚动数组）。
3. 对于高维DP，注意“维数障碍”，即状态空间过大导致无法计算。
